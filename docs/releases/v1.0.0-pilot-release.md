## Features Included

### Core Platform
- **Public Event Browsing** — Guests view upcoming events with search/filter
- **Reservation System** — Full reservation flow with guest count, contact details
- **QR Check-in** — Token-based QR codes for door staff scanning
- **Admin Dashboard** — Reservation management with status workflow (PENDING → APPROVED → REJECTED → CHECKED_IN)
- **Event CRUD** — Create, read, update, delete events via admin panel
- **Capacity Management** — Real-time enforcement with guest count validation

### Integrations
- **Google Sheets Sync** — Automatic reservation append with retry logic (5 attempts)
- **Venue Analytics** — Event attendance, reservation trends, conversion metrics
- **Health Monitoring** — `/api/health` with DB connectivity check

### Security
- **NextAuth Credential Login** — Admin authentication with JWT sessions (24h)
- **Route Protection** — Middleware guards `/admin`, `/checkin`, `/api/reservations`
- **Security Headers** — CSP, HSTS, X-Frame-Options, Referrer-Policy
- **Rate Limiting** — In-memory IP-based limiter (auth, register, mutation, public)
- **Host Validation** — Prevents host header injection attacks
- **CSRF Protection** — Origin validation on mutation requests

### Infrastructure
- **CI Pipeline** — GitHub Actions workflow for automated testing
- **Docker Support** — Multi-stage production build with standalone output
- **Vercel Deployment** — Automatic deploys from master branch
- **PostgreSQL/Neon** — Prisma ORM with connection pooling

## Known Limitations

| Issue | Impact | Workaround |
|-------|--------|------------|
| No email confirmations | Guests don't receive booking emails | Manual follow-up via admin |
| Scanner camera broken | QR scanner can't access device camera | Manual check-in via admin panel |
| In-memory sheets mapping | Sheet mappings lost on restart | Re-create mappings after deploy |
| Admin search non-functional | Can't search reservations by text | Use status filter + pagination |
| Security middleware unwired | `proxy.ts` not connected as Next.js middleware | Connect `proxy.ts` → `middleware.ts` |
| No viewport meta on guest pages | Mobile rendering not optimized | Add `<meta name="viewport">` to `_app.tsx` |
| Rate limiter is single-instance | Limits don't share across Vercel edge instances | Acceptable for pilot scale |

## Security Improvements

| Improvement | Description |
|-------------|-------------|
| Host Header Validation | Blocks requests with mismatched Host headers |
| Rate Limiting | Per-IP limits: auth (5/min), register (3/min), mutation (10/min), public (60/min) |

## Deployment Lessons

### Lesson 1: Vercel File Upload Limit
**Problem:** Initial deploy failed with "34227 files exceeds 15000 limit"
**Root Cause:** `.vercelignore` didn't exclude test files, docs, and backup directories
**Fix:** Added `__tests__/`, `docs/`, `*.md`, `node_modules_old/`, `.next.old_broken/` to `.vercelignore`
**Rule:** Always audit `git ls-files | wc -l` before Vercel deploys; keep under 15K files

### Lesson 2: Invalid Host on Production
**Problem:** All requests returned `{"error":"Invalid host"}` after deploy
**Root Cause:** `VERCEL_URL` env var is empty on Vercel production; allowlist only had `localhost:3000`
**Fix:** Allow `*.vercel.app` domains + derive host from `NEXTAUTH_URL`
**Rule:** Never rely solely on `VERCEL_URL` for host validation

### Lesson 3: Build Requires NEXTAUTH_URL
**Problem:** Local `npm run build` fails without `NEXTAUTH_URL` set
**Root Cause:** `next-auth` requires `NEXTAUTH_URL` for route configuration
**Fix:** Set `NEXTAUTH_URL` in `.env` for local development
**Rule:** Document all required env vars in `.env.example`

### Lesson 4: Vercel CLI Upload Size
**Problem:** `--archive=tgz` timed out on slow connections
**Root Cause:** Full project upload exceeds timeout threshold
**Fix:** Use `.vercelignore` to reduce file count below 15K
**Rule:** Prefer `.vercelignore` over `--archive=tgz` for large projects

## Rollback Procedure

### GitHub Rollback
```bash

git revert HEAD
git push origin master
# Vercel auto-deploys the revert commit
```

### Vercel Rollback
1. Go to Vercel Dashboard → Deployments
2. Find last working deployment (e.g., `8o2JyJXcKGFdgoRVRgMxAXnG9SPL`)
3. Click "..." → "Redeploy"
4. Or promote that deployment to production
