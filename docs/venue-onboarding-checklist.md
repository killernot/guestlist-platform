# Venue Onboarding Checklist

Use this checklist when onboarding a new venue for the pilot.

## Pre-Event

### Venue Details
- [ ] Venue name: ___________
- [ ] Venue address: ___________
- [ ] Contact person: ___________
- [ ] Contact email: ___________
- [ ] Contact mobile: ___________
- [ ] Event name: ___________
- [ ] Event date/time: ___________
- [ ] Venue capacity (fire-code limit): ___________
- [ ] Expected guest count: ___________

### Platform Setup
- [ ] Admin account created: ___________
- [ ] Staff accounts created (if needed): ___________
- [ ] Google Sheet created and linked
- [ ] Google Sheet URL: ___________
- [ ] Google Sheet permissions: service account has Editor access

### Google Sheets Configuration
- [ ] Service account email: ___________
- [ ] Google Sheets API enabled
- [ ] Google Drive API enabled
- [ ] Private key generated and stored securely
- [ ] Test: Sheet creates successfully via admin panel

### Test Reservation Flow
- [ ] Created test event: ___________
- [ ] Made test reservation: ___________
- [ ] Approved test reservation
- [ ] QR code generated: ___________
- [ ] Checked in test reservation
- [ ] Google Sheet updated: ___________
- [ ] Analytics reflected check-in: ___________

## Event Day

### Before Doors Open (T-60 min)
- [ ] Verify production URL is accessible
- [ ] Verify admin login works
- [ ] Verify scanner page loads on mobile
- [ ] Verify Google Sheet is accessible
- [ ] Verify analytics dashboard loads
- [ ] Confirm capacity settings match venue
- [ ] Pre-approve reservations (if applicable)

### During Event
- [ ] Monitor reservation approvals
- [ ] Monitor check-in flow
- [ ] Handle any issues (see Failure Procedures)
- [ ] Keep scanner device charged

### After Event
- [ ] Review analytics dashboard
- [ ] Download Google Sheet data
- [ ] Take database backup
- [ ] Archive event
- [ ] Collect feedback
- [ ] Generate post-event report
